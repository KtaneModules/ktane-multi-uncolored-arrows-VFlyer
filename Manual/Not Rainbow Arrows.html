On the Subject of Not Rainbow Arrows

What do you mean, "Point at Inverted Stores' Calculations!?" It points to nonsense!

This module has eight arrow buttons in a randomized rainbow color arrangement, and a display screen in the center showing a randomly colored number from -7812 to 7812 inclusive. The eight possible colors the module can show are red, orange, yellow, green, blue, purple/violet, white, and black.

If the expert is not prepared to do extreme levels of calculation, tell the defuser to discard this module immediately. Do not continue looking at this manual. Do not ask the defuser about it. Just. Throw it away. Burn the manual.

Examine the flashes for each stage and perform mathematical operations to the values given to submit in balanced quinary, or balanced base-5. Tilting the provided casing may assist with finding the flashes better. Each flash from the arrows will denote a function from a table on this manual that must be performed on the current value. There are 3 stages to go through with each stage after the first adding one more flash than the previous stage, and the first stage having 2 flashes before the center text changes color briefly. The module may start on a later stage, so be careful! Completing a stage WILL change the color of the text, which will require the expert to obtain a different sequence.

Each stage will utilize an initial value alongside a value that you must submit at the end of the stage. In addition, the values each colored arrow are assigned a different power of a given base that the expert must determine.

Obtaining the Calculated Value

Each arrow flash denotes an equation for each stage which the expert will need to calculate. A standard flash (1 specific color flash) will require the expert to solve for the output value, typically labeled as F(x). This is done by substituting x with an-1, bn-1, or cn-1 and solving for F(x), after applying other variables. An inverted flash (all but that specific color flash) will require the expert to solve for the input value, or x. This is also done in a similar fashion to solving for F(x) but with substituting F(x) with an-1, bn-1, or cn-1 and solving for x. The equations from the table underneath shown will not all be in F(x) = m * x + b format. Some manipulation to the equation is required.

If the calculated value after finding the given variable is:

  • greater than 7812, subtract 7813 to the value until it is 7812 or lower.
  • smaller than -7812, add 7813 to the value until it is -7812 or higher.

For the initial values:

  • a0 is the displayed value on the module.
  • b0 is the last 3 base-36 digits in the serial number, modulo 15625, minus 7812. See the manual from Unfair's Revenge for obtaining the base-36 value.
  • c0 is the first 3 base-36 digits in the serial number, modulo 15625, minus 7812. See the manual from Unfair's Revenge for obtaining the base-36 value.

Other Variable Identifications:

  • D = Sum of base 36 digits in serial number
  • n = Current step in the calculations, 1 being the first step
  • pn = the value at the specified step of the sequence. ("p" can be a, b, or c when specified, if n is not substituted, it can refer to the specific step of the calculations)

a2, b3, c4 are stages 1, 2, and 3's values to submit respectively.

Submitting The Calculated Value

In order to determine the values each colored button has, the expert will need to go through a series of condition in order to figure out which base belongs to the given colored button. Start on the rule corresponding to the color of the text to obtain the color that refers to the given value. Then use the color assigned from the last assigned arrow as the new rule to obtain the next color. Repeat until all of the colors are used up. If the target arrow is already assigned a given base or the target arrow is white or black, take the next arrow based on distance from the target arrow. If multiples on an arrow are not assigned and are tied from distance from the target arrow, assign the one going clockwise from the target color if it's not black or white. I.E. If the colors around the module are white, green, orange, purple, black, red, yellow, blue; and the target color is black, the color that should be taken instead should be red, purple otherwise if red is assigned a value.

The other arrows' ranges when calculated clockwise can be found by adding 45° to that given ranges for that given arrow.

Appendix: Converting Base-5 To Balanced Quinary

To convert a base-5 value to balanced quinary, perform the following steps from right to left on the given value:

625 125 25 5 1
1 4 0 2 3
1 4 0 2+1 3-5
1 4 0 3 -2
1+1 4-5 0 3 -2
2 -1 0 3 -2
  1. Start with the least significant digit in that set.
  2. Repeat the following procedure until there are no more significant digits to convert:
    • If the digit is a 0, 1, or 2, leave that digit alone.
    • If the digit is a 3, 4, or 5 (after adding), subtract 5 to this digit and add one to the more significant digit.
    • Go to the more significant digit and check on the top 2 statements above.
  3. This should result in your balanced quinary.

Appendix: Converting Base 10 to Base 5

  1. Start with your current base-10 value that you have.
  2. Take your current base-10 value and modulo it by 5.
  3. Assign that as your least significant digit on the module and divide your original base-10 value by 5, dropping any decimals/remainders.
  4. Repeat the previous 2 steps with your new base-10 value while adding more significant digits to each, until your base-10 value after dividing is 0.
